Compiling the INITS package

INITS is a program used to generate cosmological initial conditions for the single grid hydrodynamics code KRONOS and the adaptive mesh refinement code Enzo. See the section on obtaining Enzo for instructions on downloading inits. The routines will be in a directory called inits.

The compilation process uses a simple Makefile method in which a macro set in the first few lines of the Makefile (MACHINE_NAME), specifies a secondary Makefile (called Makefile.MACHINE_NAME) which contains machine specific information. This is very similar to the way in which Enzo is compiled. In fact the secondary Makefiles should be interchangeable.

If the machine you are running on is supported (i.e. it already has a Makefile.MACHINE_NAME file), then generally you will only have to specify the location of the HDF library (set with the LIB and INCLUDE macros). In particular, the NCSA machines should require no changes for compilation. If the secondary makefile does not exist for your machine type, then you will have to create a new one, following an existing one as a model. See also the discussion for the AMR code enzo.

Once the MACHINE_NAME macro is set, compile with:

 make

The executable inits should be created in the same directory.
